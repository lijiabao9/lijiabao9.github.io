JS两种模块化规范CommonJS与ESM

发表于2024-04-17|更新于2024-06-04|前端开发
|字数总计:834|阅读时长:3分钟|阅读量:

在 JavaScript 中，模块化开发，一些容易混淆的地方。

模块化开发，分两种规范：

  1. CommonJS，基于nodejs环境，比如 webpack、babel 等；

  2. ESM（ECMA Script Modules），基于浏览器环境，比如 html 中带 type=”module” 的 script 标签、vue 开发、react开发；

规范名称导出关键词导入关键词重命名集体导出
CommonJSmodule.exports / exportsrequire{A ：B }不解构
ESMexportimportA as B* as XXX

CommonJS 规范

CommonJS 规范是为了解决 JavaScript 的作用域问题而定义的模块形式，可以使每个模块在它自身的命名空间中执行。该规范的主要内容是，模块必须通过 module.exports 导出对外的变量或接口，通过 require() 来导入其他模块的输出到当前模块作用域中。

为了方便，某些情况下 module.exports 可以简写为 exports。
其实一开始 exports 就是 module.exports 的引用。
所以不要对 exports 重新赋值，不然此变量指向其他的引用，就导出不了。

导出的变量，避免可能出现的命名冲突，可以用( ：)符号重命名。

除了解构写法，还可以用一个变量接收全部导出。

ESM（ECMA Script Modules）规范

ESM 是将 javascript 程序拆分成多个单独模块，并能按需导入的标准。和 webpack，babel 不同的是，esm 是 javascript 的标准功能，在浏览器端和 nodejs 中都已得到实现。使用 esm 的好处是浏览器可以最优化加载模块，比使用库更有效率。通过 import, export 语法实现模块变量的导入和导出。

esm 的 export 支持：let、var、const、function、class 等

想要导出所有方法或变量为指定的一个变量时，可以用 import * as foo from "bar"

和 CommonJS 规范先进一点的是 ESM 提供了一种默认导出的方式。

export default 后面可以直接跟随：变量/常量、function、calss。
