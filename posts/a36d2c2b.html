自动化运维：使用 acme.sh 配合 Cloudflare 申请免费泛域名证书

发表于2026-05-24|更新于2026-05-31|DevOps域名管理
|总字数:1.2k|阅读时长:4分钟|浏览量:

为了保障数据传输安全，实现全站 HTTPS，我们需要为域名申请 SSL 证书。

本文将指导你使用业内知名的自动化证书管理工具 acme.sh，通过 Cloudflare 的 DNS-01 验证方式，申请由 Let’s Encrypt 签发的免费 ECC 算法泛域名证书。

泛域名证书（如 *.131495.xyz）可以同时保护你的主域名和所有二级子域名，且整个续期流程完全自动化。

一、 获取 Cloudflare 专属 API Token

为了让 acme.sh 能够自动在 Cloudflare 中添加用于验证所有权的 TXT 记录，我们需要为其创建一个最小权限的 API 令牌（Token）。

  1. 登录 Cloudflare 官网
  2. 在左侧菜单中选择 管理账户 -> 账户 API 令牌，点击 创建令牌
  3. 配置权限：
    • 令牌名称：例如命名为 用于 ACME 自动续期的令牌
    • 权限策略：选择 Edit Zone DNS，表示可编辑所有域名的 DNS 记录
    • 令牌过期时间：选择 无过期时间
    • 客户端 IP 地址过滤：可配置 IP 黑白名单
  4. 点击 审核令牌 并确认 创建令牌。复制并妥善保存生成的 Token，该 Token 仅显示一次。

四、 申请 ECC 泛域名证书

相较于传统的 RSA 算法，ECC 算法证书（通常为 256 位密钥）拥有更小的体积、更快的握手速度以及更低的 CPU 消耗，是现代生产环境的首选。

五、 正确安装证书与自动化重载

高危规范警告：
切勿直接在例如 Nginx 配置文件中引用 ~/.acme.sh/ 目录下的原始证书文件！
该目录属于 acme.sh 的内部工作区，其结构在未来升级时可能会发生变化。
正确且规范的做法是使用 --install-cert 命令将证书分发至专用的生产环境目录，并注册服务重载事件。
