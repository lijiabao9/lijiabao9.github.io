镜像构建

业务场景：开发好的 JAVA 程序，可以将其打包成一个镜像，后续部署就不用额外配置环境了。

所谓镜像，就是包含了应用程序、程序运行的系统函数库、运行配置等文件的文件包。
构建镜像的过程，就是把上述文件打包的过程。

容器网络

默认情况下，所有的容器，都是以 bridge 方式连接到 docker 的一个虚拟网桥上，

所有的容器，都会分配到一个ip地址，容器间可以通过这些 ip 实现互相访问。

但是容器 ip 是动态的，每次创建/重启都会重新动态分配。

docker 允许自由声明网络，加入同一个网络的容器间，可以通过 容器名称 互相访问。

容器编排

对于一个普通的应用，一般都需要部署多个容器，比如前端 nginx，后端 jar，数据库 mysql。

如果手动一个一个去部署，会很繁琐。

docker 提供了一种容器编排技术，DockerCompose。

DockerCompose，通过一个单独的 docker-compose.yml 文件来定义一组相关联的应用容器，
帮助我们实现多个相互关联的 Docker 容器的快速部署，以及统一管理。

配置文件的本质，可以看作是 docker run 命令的映射，告诉 docker 怎样创建容器，以及容器之间的关系（如依赖）。
