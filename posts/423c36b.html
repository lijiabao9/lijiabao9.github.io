域名无缝托管至 Cloudflare 全指南

发表于2026-05-24|更新于2026-06-30|域名管理
|总字数:543|阅读时长:1分钟|浏览量:

国内域名解析面临实名认证、备案等繁琐流程，且在全球加速与安全防护上与国际标准有差距。

推荐将 DNS 解析权移交至 Cloudflare，获得顶级加速和防御保护。本文以 SpaceShip 购买的 .xyz 域名为例。

1. 接入 Cloudflare 站点

  1. 登录 Cloudflare → 域名 → 概览 → 添加域名 → 连接域名
  2. 输入域名（如 example.xyz）→ 继续
  3. 计划选择 Free（免费计划）
  4. Cloudflare 自动扫描现有 DNS 记录 → 继续

2. 修改 NameServer

Cloudflare 会分配两个 NS 地址（如 alina.ns.cloudflare.com 和 tim.ns.cloudflare.com），需到原注册商替换。

  1. SpaceShip 控制台 → Domain Manager → 点击域名
  2. NameServers & DNS → Nameservers → Change
  3. 类型改为 Custom nameservers
  4. 填入 Cloudflare 提供的两个 NS 地址，保存

全球 DNS 刷新需几分钟到几小时，收到 Cloudflare 邮件 Status: Active 即托管成功。

    graph LR
    Browser[访问者] -->|解析请求| SpaceShip[SpaceShip 注册商]
    SpaceShip -->|重定向 NS| CF_NS[Cloudflare 智能解析]
    CF_NS -->|返回加速IP| Browser

3. DNS 记录与代理状态

托管成功后，所有解析记录在 Cloudflare 域名详情的 DNS → 记录 中配置。

  • A 记录：域名指向服务器公网 IPv4
  • CNAME 记录：别名解析（如 www 指向主域名）

代理状态（小黄云）

状态 说明
开启（橙色云朵） 流量经 Cloudflare CDN，隐藏真实 IP，提供免费 SSL 和 DDoS 防护
关闭（灰色云朵） 仅作纯 DNS 解析，流量直连服务器，暴露真实 IP

4. 生产级优化配置

SSL/TLS

SSL/TLS → 概述：设置为 完全（严格）

要求源服务器也配置有效 SSL 证书，是最安全的防劫持加密模式。

速度与网络

  • 速度 → 配置：启用建议中的所有可用配置（Speed Brain、Cloudflare Fonts、Early Hints、0-RTT 连接恢复）
  • 网络：开启 gRPC

规则设置

规则 → 设置：

配置 操作
添加 TLS 客户端身份验证标头 开启
添加访问者位置标头 开启
添加泄露凭据检查标头 开启
删除 X-Powered-By 标头 开启（隐藏后端技术栈）
删除访问者 IP 标头 关闭
添加安全性标头 关闭
URL 规范化 开启
