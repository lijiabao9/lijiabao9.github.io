玩转 Microsoft 365 E3：低成本搭建高权重企业级 SMTP

发表于2026-05-24|更新于2026-05-26|DevOps运维手册
|总字数:2.3k|阅读时长:7分钟|浏览量:

拥有 Microsoft 365 E3 订阅后，我们不仅获得了全套 Office 生产力工具，还解锁了企业级的 Exchange Online 服务。

相比于个人版 Outlook，企业版 Exchange 允许我们绑定自定义域名（如 admin@gabriel.edu.kg），并提供极高的发信权重。

本文将指导你将其配置为高送达率的 SMTP 服务，用于博客通知、Alist 消息推送、Uptime Kuma 告警等场景。

第一步：域名绑定与安全加固

1. 绑定自定义域名

为了让发出的邮件显示你的专属域名（而不是自带的 xxx.onmicrosoft.com），我们需要先在全局中绑定域名。

  1. 进入管理中心：访问 M365 Admin Center，点击左侧导航栏的 Settings（设置） -> Domains（域）。

  2. 添加域名：点击 Add domain，输入你拥有的域名（如 gabriel.edu.kg）。

  3. 验证域名：

    • 自动验证（推荐）：如果你的域名托管在 Cloudflare、GoDaddy 等主流平台，选择 “Sign in to…” 即可授权自动添加 DNS 记录。

    • 手动验证：选择 “Add a TXT record to…”，将生成的 TXT 记录手动复制到你的 DNS 解析服务商处。

  4. 配置 DNS 记录：验证通过后，系统会要求配置 Exchange 的 DNS 记录。

    • 建议勾选 Exchange 相关的所有基本记录（MX, CNAME, TXT）。

    • 按照提示在 DNS 服务商处添加对应的解析记录，等待状态显示为 Healthy 即代表成功。

2. 域名配置 DKIM 认证（防止邮件进垃圾箱）

必须配置：为了防止 Gmail、Outlook 等收件方将你的通知邮件直接归类到垃圾邮件箱，必须配置 DKIM（域名密钥识别邮件）。

  1. 访问 Defender 防护中心：
    点击跳转 DKIM 配置页面

  2. 启用 DKIM：

    • 在列表中点击你的域名 gabriel.edu.kg。

    • 将 Sign messages for this domain with DKIM signatures 的滑块切换为 Enable (开启)。

  3. 添加 CNAME 记录：

    • 如果系统提示 “You need to create CNAME records…”，请按照提示在你的 DNS 服务商处添加 2 条 CNAME 记录。

    • 记录名通常为 selector1._domainkey 格式。

  4. 再次启用：

    • DNS 生效后（通常几分钟），回到此页面再次点击开启滑块。

    • 直到状态显示为绿色的 Status: Enabled。

第二步：创建专用发信账户

1. 新建服务账户（Service Account）

为了确保主账号的安全，强烈建议不要直接使用全局管理员账号进行 SMTP 发信。我们需要创建一个权限受限的专用账户。

  1. 访问 M365 Admin Center，点击 Users（用户） -> Active users（活跃用户）。

  2. 点击 Add a user：

    • First/Last name： 建议填写 Notification Bot。

    • Username：建议使用 noreply。

    • Domain：下拉菜单中务必选择刚才绑定的自定义域名 gabriel.edu.kg。

    • Password：手动设置或自动生成，请务必妥善记录此密码。

    • Licenses：必须勾选包含 Exchange 服务的订阅（如 Office 365 E3 Developer），否则该账户将不具备邮箱功能。

2. 激活 Authenticated SMTP 权限

新创建的用户默认处于安全考虑，可能未开启传统 SMTP 传输权限，需要手动启用。

  1. 在用户列表中，点击刚才创建的 noreply 用户。

  2. 在右侧弹出的详情面板中，切换到 Mail（邮件） 选项卡。

  3. 点击 Manage email apps（管理电子邮件应用）。

  4. 勾选最底部的 Authenticated SMTP（经过身份验证的 SMTP），并保存修改。

微软全球服务器同步可能需要时间：
这里的权限修改通常需要 15 ~ 60 分钟 才能完全在全球节点生效。如果配置完后立即测试连接遭遇失败，请耐心等待后再试。

方案一：配置传统 SMTP 发信（基础认证）

1. 关闭全局安全默认值

Microsoft Entra ID（原 Azure AD）默认启用的安全策略会全局阻止传统 SMTP 基础认证。
如果不关闭此选项，后续的 SMTP 连接握手将持续报错。

  1. 访问 Microsoft Entra Admin（需使用 E3 全局管理员账号登录）。

  2. 点击 Manage Microsoft Entra ID 卡片的 View 按钮。

  3. 上方菜单栏中选择 Properties（属性）。

  4. 滚动到页面最底部，点击蓝色链接 Manage security defaults（管理安全默认值）。

  5. 在右侧弹出的面板中，将 Security defaults 切换为 Disabled（禁用）。

    • Reason for disabling：可随意选择，例如 “Application requires legacy authentication”。

  6. 点击 Save 保存。

2. SMTP 配置参数速查表

现在，你可以将以下标准参数填入你的博客（WordPress、Artalk）、监控（Uptime Kuma）或网盘（Alist）等应用的邮件通知设置中：

配置项 推荐值 说明
SMTP Host smtp.office365.com 微软官方统一发信服务器地址
SMTP Port 587 推荐端口（非 SSL 加密端口 465）
Encryption STARTTLS 或 TLS 必须选择安全传输协议
Username noreply@gabriel.edu.kg 你创建的完整发信邮箱地址
Password xxxxxxxx 该专用发信用户的登录密码

方案二：配置 Azure 应用（基于现代 OAuth2 / Graph API）

如果你使用的第三方应用（如 Apprise, LobeChat, Markdown 自动化脚本）支持 OAuth2 认证，或者支持通过官方 Microsoft Graph API 发信，这是更安全、不依赖关闭安全默认值的现代发信方案。

1. 注册 Azure 应用

  1. 访问 Microsoft Entra Admin（需使用 E3 全局管理员账号登录）。

  2. 点击 Manage Microsoft Entra ID 卡片的 View 按钮。

  3. 左侧导航栏中，选择 Manage -> App registrations（应用程序注册）。

  4. 注册应用程序，点击上方的 New registration（新注册)。

    • Name：填写应用名称（例如 M365-Mail-Bot）

    • 其余选项保持默认，点击 Register 完成注册

  5. 点击应用名称，进入应用详情页，复制并保存以下两个核心凭证：

    • Application (client) ID

    • Directory (tenant) ID

2. 应用配置 API 权限

  1. 在应用详情页。

  2. 左侧导航栏，选择 Manage -> API permissions（API 权限）

  3. 添加权限，点击 Add a permission，选择 Microsoft Graph，选择 Application permissions（应用程序权限）

  4. 在弹出的搜索框中，输入并勾选以下权限：

    • Mail.Send（允许应用发送邮件）

    • Mail.ReadWrite（可选，用于处理草稿箱或发件箱）

  5. 点击 Add permissions 完成添加权限。

  6. 刷新授权。回到权限列表，此时状态会显示 “Not granted for…”

    点击列表上方醒目的 Grant admin consent for <组织名>（授予管理员同意） 按钮并确认。

    看到 Status（状态列）转为绿色的 Granted for xxx 即代表 API 授权成功。

3. 生成一个客户端密钥（Client Secret）

  1. 在应用详情页。

  2. 点击左侧导航栏的 Manage -> Certificates & secrets（证书和密码）。

  3. 选择 Client secrets 选项卡，点击 New client secret。

    • Description：任意填写密钥描述信息

    • Expires：选择一个合适的有效期

  4. 点击 Add 添加后，务必立即复制 “Value” 字段的值（该值在刷新或离开页面后将永久隐藏）。

    • 注：若生成的密钥中包含特殊字符，在某些应用中配置时可能需要进行 URL Encode。

此时，基于 Application (client) ID、Directory (tenant) ID、Client Secret，以及任意发信账户（无需密码）。
你可以在第三方应用中配置基于 OAuth2 的授权，并使用 Graph API 发信。

进阶玩法：配置 Catch-all 规则实现“无限匿名邮箱”

除了基础发信，Exchange Online 还提供了极其强大的路由规则。

通过配置 Catch-all，你可以随意编造任意后缀的临时邮箱（如 amazon-reg@gabriel.edu.kg）用于注册网站，所有发往这些不存在账户的邮件都会被自动汇聚到你的主邮箱中。

1. 修改域名接受类型

  1. 访问 Microsoft Exchange Admin。

  2. 左侧导航栏，点击 Mail flow（邮件流） -> Accepted domains（接受的域）。

  3. 点击你 gabriel.edu.kg 进入详情页。

  4. 将 This accepted domain is 选项从 Authoritative（权威）切换为 Internal relay（内部中继），然后点击 Save。

2. 创建邮件转发规则

  1. 左侧导航栏，点击 Mail flow（邮件流） -> Rules（规则）。

  2. 创建新规则，点击 Add a rule -> Create a new rule。

    • Name：填写规则名称，如 Catch-all-Forwarder。

    • Apply this rule if（满足以下条件时应用）：选择 收件人邮箱地址满足格式 The recipient -> address matches any of these text patterns。

      • specify text patterns：输入你希望匹配的邮箱规则正则表达式。

      • 例如：输入 ^test.*@gabriel\.edu\.kg$ 可以精准匹配所有以 test 开头的邮箱。

    • Do the following（执行以下操作）：

      • 选择 邮件重定向至 Redirect the message to -> these recipients，并在弹出的组织用户列表中选择你的主账号。

  3. 后续的高级设置保持默认，点击 Finish 完成创建。

3. 启用规则

新创建的邮件流规则默认处于 Disabled（禁用） 状态。

请在规则列表中点击该规则，在右侧弹出的面板中将状态滑块切换为 Enabled。

配置完成：
现在，你可以直接在外网使用 test-any-string@gabriel.edu.kg 进行服务注册了。
所有发往该地址的验证码与通知，都会被 Exchange 完美重定向投递到你的主收件箱中，既保护了真实主账号，又实现了无限别名。
