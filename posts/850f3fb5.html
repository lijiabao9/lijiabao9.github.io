Windows 环境下 PostgreSQL 绿色压缩包版安装与 PostGIS 空间扩展集成指南

发表于2026-05-23|更新于2026-05-28|开发环境Windows
|总字数:1.4k|阅读时长:4分钟|浏览量:

相比于传统的 EXE 安装程序，使用 ZIP 压缩包 方式部署 PostgreSQL 更加清爽高效：
它不会在注册表中留下难以清理的残留垃圾，且天然支持多版本完美共存，非常适合开发与测试环境。

本文将为你演示在 Windows 环境下从零开始部署 PostgreSQL 18.4 绿色版，并深度集成 PostGIS 空间地理信息扩展的完整流程。

一、 PostgreSQL 基础安装

1. 获取官方压缩包

前往 EnterpriseDB 官方二进制包下载页面（或 PostgreSQL 官网的 Binaries 页面）：

官方下载地址：https://www.enterprisedb.com/download-postgresql-binaries

将下载好的压缩包解压到你指定的非系统盘目录下（路径中尽量避免包含中文或特殊字符）。

  • 示例路径：D:\App\PostgreSQL\postgresql-18.4-1\

2. 配置全局环境变量（强烈推荐）

为了避免每次使用 psql 或 pg_ctl 等核心工具时都需要繁琐地切换目录，建议优先配置系统环境变量：

  1. 右键点击 此电脑 -> 选择 属性 -> 点击 高级系统设置 -> 点击 环境变量。
  2. 在 系统变量 区域中找到并选中 Path 变量，点击 编辑。
  3. 点击 新建，将你的 PostgreSQL bin 目录完整路径添加进去：D:\App\PostgreSQL\postgresql-18.4-1\bin。
  4. 一路点击 确定 保存退出。

3. 初始化数据库

以 管理员身份 打开命令提示符（CMD）或 PowerShell，执行数据库集群的初始化命令：

💡 核心参数详解：

  • -D "..."：指定数据存储目录（该目录会自动生成，切勿手动创建）。
  • -U postgres：指定数据库超级管理员用户名为 postgres。
  • -A password：指定本地连接的认证方式为密码认证（加密强度高）。
  • -E UTF8：设置数据库默认编码为全球通用的 UTF-8。
  • --locale=C：设置区域语言环境为 C（有效避免因操作系统本地语言导致的排序错乱或日志乱码）。
  • -W：强制在控制台提示输入并确认超级用户的初始密码。

⚠️ 注意：命令回车后，控制台会提示你输入两次密码，请务必牢记该密码，它将作为你的管理员登录密码！

二、 PostGIS 空间扩展集成

PostGIS 是 PostgreSQL 的“灵魂伴侣”，集成后将使数据库获得强大的空间地理信息（GIS）存储、索引和分析计算能力。

1. 下载插件包

前往 OSGeo 下载对应版本的 PostGIS 编译压缩包。请务必注意与你的 PostgreSQL 大版本号保持对应（如对应 pg18）：

官方下载地址：https://download.osgeo.org/postgis/windows/

2. 手动覆盖安装（最稳妥的方式）

虽然 PostGIS 压缩包内有时会附带安装脚本，但由于路径识别极易因系统差异而报错。推荐使用最纯粹的“文件夹合并覆盖法”：

  1. 解压下载好的 PostGIS 压缩包。
  2. 进入解压后的文件夹，将里面的 bin、lib、share、gdal-data 这 4 个核心文件夹全选并复制。
  3. 直接粘贴到 PostgreSQL 的安装根目录下（即 D:\App\PostgreSQL\postgresql-18.4-1\）。
  4. 系统若提示文件冲突或重复，请果断选择 “替换目标中的文件”。
