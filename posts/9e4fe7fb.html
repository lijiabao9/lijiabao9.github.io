SpringBoot 中 MyBatis-Plus 的使用

发表于2025-10-17|更新于2025-11-06|编程代码
|总字数:3.6k|阅读时长:14分钟|浏览量:

MyBatis-Plus 是一个 MyBatis 增强工具，用于简化开发，提高效率。

官网地址：https://baomidou.com/getting-started

快速开始

引入依赖

定义 Mapper 接口

MyBatis-Plus 提供了一个基础的 BaseMapper 接口，继承该接口即可完成对数据库的增删改查操作。

提供了 insert、delete、update、select 等一系列方法。

需要在接口中添加注解 @org.apache.ibatis.annotations.Mapper。
或者在启动类中添加 @org.mybatis.spring.annotation.MapperScan("com.example.demo.mapper")

实体类注解

实体类与数据库表格的对应关系，MyBatis-Plus 遵守以下约定：

  • Entity 的类名，驼峰转下划线，作为表名

  • Entity 的属性，驼峰转下划线，作为字段名，根据属性类型推断字段类型

  • 约定 id 为主键字段，类型为 Long

与上述约定不符的，提供了注解进行自定义配置：

常用的全局配置

持久层接口

Service 层

MyBatis-Plus 提供了 IService 接口，继承该接口即可完成对数据库的增删改查操作，此接口的默认实现为 ServiceImpl。

提供了 save、saveOrUpdate、remove、update、
get、list、page、count 等一系列方法。

Mapper 层

见 快速开始 章节。

Entity 层

ActiveRecord 模式是一种设计模式，它允许实体类直接与数据库进行交互，实体类既是领域模型又是数据访问对象。

在 Mybatis-Plus 中，实体类只需继承 Model 类即可获得强大的 CRUD 操作能力。

提供了 insert、delete、update、select 等一系列方法。

Service 层的 chain 操作

在 Service 层，提供了 chain 操作，用于链式操作，可以连续调用方法来构建查询条件或更新条件。

SimpleQuery

SimpleQuery 是 Mybatis-Plus 提供的一个工具类，它对 selectList 查询后的结果进行了封装，
使其可以通过 Stream 流的方式进行处理，从而简化了 API 的调用。

Db Kit

Db Kit 是 Mybatis-Plus 提供的一个工具类，它允许开发者通过静态调用的方式执行 CRUD 操作，
从而避免了在 Spring 环境下可能出现的 Service 循环注入问题，简化了代码，提升了开发效率。

使用方法和 Service 层的方法类似，只是多了 Entity 类型的指定。

条件构造器 Wrapper

基本用法

MyBatis-Plus 提供了一套强大的条件构造器（Wrapper），用于构建复杂的数据库查询条件。
Wrapper 类允许开发者以链式调用的方式构造查询条件，无需编写繁琐的 SQL 语句，从而提高开发效率并减少 SQL 注入的风险。

上述的条件构造器，在使用字段名的过程中，会出现魔法值的情况。
因此 MyBatis-Plus 又提供了一套基于 Lambda 的 Wrapper。

Wrappers

MyBatis-Plus 提供了 Wrappers 类，它是一个静态工厂类，
用于快速创建 QueryWrapper、UpdateWrapper、LambdaQueryWrapper 和 LambdaUpdateWrapper 的实例。
使用 Wrappers 可以减少代码量，提高开发效率。

使用 Wrapper 自定义 SQL

MyBatis-Plus 提供了强大的 Wrapper 条件构造器，允许开发者自定义 SQL 语句，以满足更复杂的数据库查询需求。

在 service 层构造 wrapper 对象，并传入自定义 SQL 片段，即可实现自定义 SQL 的查询。

自动映射

自动映射枚举

假设 User 有一个字段 gender，表示性别，1-男性，2-女性。

对于这种字典取值，常常会定义一个枚举类。
JAVA 中实体的属性类型为该枚举类，然后数据库中存储是 int 类型。
对于这种转换处理，MyBatis-Plus 提供了 EnumTypeHandler 枚举处理器，

Mybatis-Plus 提供了 @EnumValue 注解，标记枚举类的属性为数据库字段的值；
Jackson 框架提供了 @JsonValue 注解，标记枚举类的属性为 JSON 字段的值；

自动映射 JSON

假设 User 有一个字段 info，表示用户信息，是一个 JSON 字符串。

在 JAVA 的实体中，可以将属性定义为 String 类型。但是后续的处理会很麻烦，考虑将其定义为新的类。

MyBatis-Plus 提供了 AbstractJsonTypeHandler 枚举处理器，用于处理此类字段转换。

特性功能

自动填充字段

MyBatis-Plus 提供了一个便捷的自动填充功能，用于在插入或更新数据时自动填充某些字段，如创建时间、更新时间等。

实体类中添加填充的字段，并使用 @TableField 注解指定填充方式。

创建一个类来实现 com.baomidou.mybatisplus.core.handlers.MetaObjectHandler 接口，并重写 insertFill 和 updateFill 方法。

逻辑删除支持

Mybatis-Plus 支持逻辑删除，即数据不会被真实删除，而是将数据标记为删除状态。

删除操作，会自动变成 update 语句，将字段 deleted 设置为 1。

查询、更新操作，会自动在 sql 语句，添加 where 条件， deleted 值为 0 的数据进行过滤。

这些动作，都是 MyBatis-Plus 自动完成的，对于开发者来说，使用语法丝毫没有变化。

在实体的逻辑删除字段上，添加注解 @TableLogic 即可；

注意：逻辑删除，与唯一索引会有冲突。可考虑将 deleted 已删除设置为 UNIX_TIMESTAMP() 时间戳。

分页插件

MyBatis-Plus 的分页插件 PaginationInnerInterceptor 提供了强大的分页功能，支持多种数据库，使得分页查询变得简单高效。

在未引入分页插件的情况下，Mybatis-Plus 是不支持分页功能的，即 IService 和 BaseMapper 中的分页方法都无法正常起效。

Page 类继承了 IPage 类，实现了简单分页模型。

乐观锁插件

乐观锁是一种并发控制机制，用于确保在更新记录时，该记录未被其他事务修改。
MyBatis-Plus 提供了 OptimisticLockerInnerInterceptor 插件，使得在应用中实现乐观锁变得简单。

在实体类中，需要在表示 版本号 的字段上添加 @Version 注解。
支持的数据类型包括：int, Integer, long, Long, Date, Timestamp, LocalDateTime。

后续的 update 方法会自动将版本号加 1，且 where 条件中会添加版本号条件。

多租户插件

TenantLineInnerInterceptor 是 MyBatis-Plus 提供的一个插件，用于实现多租户的数据隔离。
通过这个插件，可以确保每个租户只能访问自己的数据，从而实现数据的安全隔离。

本质就是 SQL 拦截，在 SQL 中添加租户字段的查询条件。

注意：表格需要有 tenant_id 字段。
注意：插入数据时，需要配置 自动填充字段 填充 tenant_id 值。

数据权限插件

DataPermissionInterceptor 是 MyBatis-Plus 提供的一个插件，用于实现数据权限控制。
它通过拦截执行的 SQL 语句，并动态拼接权限相关的 SQL 片段，来实现对用户数据访问的控制。

动态表名插件

在数据库应用程序开发中，我们有时需要根据不同的条件查询不同的表。

MyBatis-Plus 提供了一个动态表名插件 DynamicTableNameInnerInterceptor。
它可以在执行 SQL 语句之前，根据配置的规则动态地替换表名。
这个功能在处理分表逻辑时非常有用，比如根据日期将数据存储在不同的表中。

防全表更新与删除插件

BlockAttackInnerInterceptor 是 MyBatis-Plus 框架提供的一个安全插件，专门用于防止恶意的全表更新和删除操作。
该插件通过拦截 update 和 delete 语句，确保这些操作不会无意中影响到整个数据表，从而保护数据的完整性和安全性。
