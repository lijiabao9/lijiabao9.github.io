二、高级功能定制

1. 界面个性化

登录 Artalk 后台，进入 设置 -> 界面配置，可以对评论区外观和交互进行个性化调整：

  • 评论框占位文字：让思绪在此停泊，或如风，或如诗…
  • 无评论显示文字：此处静默，却等待你的第一笔心动。
  • 发送按钮文字：寄往光的深处
  • 表情包自定义：
    • 将默认的：https://cdn.jsdelivr.net/gh/ArtalkJS/Emoticons/grps/default.json
    • 修改替换为：https://cdn.jsdelivr.net/gh/eryajf/emotion_generate/dist/artalk.json
  • 功能开关：
    • 默认勾选 投票按钮
    • 默认不勾选 反对按钮
    • 勾选 用户UA徽标（开启后评论区会优雅地显示类似 Edge 137, Windows 11 的浏览器与系统图标）

2. IP 属地显示

  1. 下载离线 IP 数据库 ip2region.xdb 到 Artalk 的数据目录中：

  2. 在 Artalk 后台 -> 设置 -> IP属地，勾选 开启 IP 属地展示。

  3. 注意（针对 CDN 环境）：如果你的网站或服务端后续使用了 CDN 代理

    请在 Artalk 后台 -> 设置 -> 服务器 -> 代理标头名 中，填写 X-Forwarded-For，以确保能够正确识别访客的真实 IP。

5. 邮件通知

Artalk 支持通过 SMTP 协议 向用户投递回复通知。

在 Artalk 后台 -> 设置 -> 邮件通知，勾选 启用邮件通知：

  • 发送方式：smtp
  • 发送人昵称：{{reply_nick}}
  • 发送人地址：noreply@example.com（注意需要改为和下方配置的实际发件邮箱一致，否则可能会发送失败）
  • 邮件标题：[{{site_name}}] 您收到了来自 @{{reply_nick}} 的回复
  • 邮件模板文件：/data/mail_tpl/Typewriter.html（填写模板 HTML 的容器内路径，替换默认的 default）

在 SMTP 发送 配置项中，填写发件账号的鉴权信息：

  • 针对 QQ 邮箱：
  • 发件地址：smtp.qq.com
  • 发件端口：587
  • 用户名：your_email@qq.com
  • 密码：your_email_authcode（注意：是邮箱额外生成的 SMTP 授权码，非登录密码）
  • 针对 Microsoft 邮箱：
  • 发件地址：smtp.office365.com
  • 发件端口：587
  • 用户名：your_email@xxx.onmicrosoft.com
  • 密码：your_email_password（直接填写账户登录密码）

6. 社交登录

Artalk 默认只需填写昵称和邮箱即可发表评论，无需验证邮箱。

但有时候，我们希望用户能够使用社交账号登录，以减少用户填写信息的时间，或者提高用户信息的真实性，可以通过启用社交登录来实现这一目的。

在 Artalk 后台，设置 -> 社交登录，勾选 启用社交登录，建议也开启 允许匿名评论。
