SpringBoot 整合 MyBatis-Plus 使用教程

发表于2026-05-23|更新于2026-05-31|编程语言JAVA
|总字数:2.7k|阅读时长:11分钟|浏览量:

MyBatis-Plus (MP) 是一个 MyBatis 的增强工具，在 MyBatis 的基础上只做增强不做改变，为简化开发、提高效率而诞生。

掌握 MyBatis-Plus，能让你摆脱繁琐的单表常规 CRUD 编写，通过内置的通用接口与丰富的插件体系，将核心精力无缝聚焦于业务领域模型的构建。

MyBatis-Plus 默认遵守 “约定优于配置” 的核心原则：

  • 类名 -> 数据库表名 (自动由驼峰转下划线)
  • 属性名 -> 表字段名 (自动由驼峰转下划线)
  • 名为 id 的属性 -> 表主键

二、 核心功能详解

1. Service 层通用 CRUD

通过继承 IService<T> 和 ServiceImpl<M, T>，不仅能无缝代理 Mapper 的单表操作，更能获得批量插入、批量更新等扩展服务：

2. 响应式链式调用 (Chain)

MyBatis-Plus 提供了类似 Java Stream 流的响应式链式操作，大幅提升了代码段的可读性与流畅度。

3. ActiveRecord 模式

使实体类本身具备完备的自驱 CRUD 能力（实体对象需继承 Model<T>）。

4. Db Kit 互斥工具类

为了在复杂的业务交互中避免 Service 层之间频繁循环注入导致的系统死锁风险，MyBatis-Plus 提供了静态 Db 增强工具类。

三、 条件构造器 (Wrapper)

Wrapper 是 MyBatis-Plus 动态 SQL 组装的精髓，用于在 Java 代码层构建复杂的 WHERE 过滤条件。

2. 全局逻辑删除 (Logic Delete)

可通过全局（mybatis-plus.global-config.db-config.logic-delete-field: xxx）统一配置逻辑删除字段，或在单个实体类上通过注解（@TableLogic）单独配置。

配置逻辑删除后，底层的 deleteById 等删除方法，会被拦截并无缝重构为非破坏性的类似 UPDATE t_user SET deleted=1 WHERE id=? 更新指令。

⚠️ 生产避坑红线风险提示：
逻辑删除与数据库底层的 唯一索引 (Unique Index) 存在天然的冲突风险（例如：删除了用户名叫 jack 的记录后，再次注册 jack 会因为冲突而无法入库）。
在架构设计时，建议逻辑删除字段使用 delete_time 毫秒时间戳代替，或者将 deleted 状态字段强行包含在联合唯一索引之中。
