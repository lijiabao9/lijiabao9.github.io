基于 GitHub + jsDelivr + PicGo 搭建完全免费、极速、稳定的个人独立图床指南

发表于2026-05-23|更新于2026-05-26|DevOps效率工具
|总字数:1.8k|阅读时长:5分钟|浏览量:

在 Markdown 写作、跨平台分发或独立博客（如 Hexo、Hugo）的搭建过程中，图片的存储、托管与引用一直是创作者的痛点。

将图片直接存放在本地会导致分发困难（别人无法直接读取本地路径），而使用商业图床又随时面临收费、限流甚至运营商“跑路”的潜在风险。

本文将带你手把手搭建一套业内公认的 “GitHub（云端存储） + jsDelivr（全球加速） + PicGo（一键上传）” 黄金组合，彻底实现免费、去中心化、且国内访问极速的终身图床服务。

第一步：创建 GitHub 远程存储仓库

所谓的“图床”，本质上就是一个用来公开托管多媒体文件的云端容器。我们首先需要在 GitHub 上开辟这个空间。

  1. 登录 GitHub 官网，点击右上角的 + 号，选择 New repository（新建仓库）。
  2. Repository name：填写你的图床仓库名（例如：pic.cloud）。
  3. Visibility：必须选择 Public（公开）。如果错选为 Private（私有），后续外部的 CDN 加速节点将彻底无法读取和抓取其中的图片。

此时，你已经拥有了一个无限容量的云端存储空间。

第二步：获取 GitHub 专属安全访问令牌 (Token)

为了让第三方软件 PicGo 有权读取并直接向你的 GitHub 仓库推送文件，我们需要生成一个专属的个人访问令牌（Personal Access Token）。

  1. 点击 GitHub 右上角个人头像 -> 选择 Settings。
  2. 在左侧菜单栏的最下方，找到并点击 Developer settings。
  3. 依次点击 Personal access tokens -> Tokens (classic)。
  4. 点击右上角的 Generate new token -> 选择 Generate new token (classic)。
  5. 精细化配置 Token 参数：
    • Note (名称)：随意填写，用于标识此 Token 的用途，如 PicGo-Upload。
    • Expiration (过期时间)：为了保证本地写作流程不中断，建议选择 No expiration（永不过期）。
    • Scopes (权限勾选)：必须勾选 repo 这一根节点权限。这代表授予了 PicGo 往仓库读写文件的核心权限。
  6. 点击底部的 Generate token 按钮。

⚠️ 神级安全警告：
页面跳转后生成的 Token 字符串 只会完整显示一次！
请务必立即将其复制并妥善保存到你的本地备忘录中。
一旦刷新页面，你将再也无法看到它，只能删掉重新生成。

第三步：安装与高能配置 PicGo 客户端

PicGo 是一款目前极为优秀的开源跨平台图片一键上传利器。

1. 下载安装

前往 PicGo GitHub Releases 页面，下载适合 Windows 平台的最新稳定版安装包（例如 PicGo-Setup-2.5.3-x64.exe），双击按照默认提示完成安装。

2. 深度绑定 GitHub 图床

打开 PicGo 客户端主界面，在左侧边栏找到 图床设置 -> GitHub（如果在左侧没有看到 GitHub，请先前往“PicGo 设置”中，在“显示的图床”列表中将其勾选激活）。

点击 编辑 按钮，按照下表规范输入你的图床核心配置：

配置字段 核心说明与填写示例 是否必填
图床配置名 随意填写，用于本地多配置区分（例如：我的主图床） 是
设定仓库名 必须采用标准格式：GitHub用户名/仓库名。
例：lijiabao9/pic.cloud		 是（极易写错）
设定分支名 填入你仓库的主分支名。新版 GitHub 默认新建仓库均为 main。
例：main		 是
设定 Token 粘贴你在第二步中保存的以 ghp_ 开头的长字符串 是
设定存储路径 图片在仓库中的子目录路径。绝对不能以 / 开头，但必须以 / 结尾。
例：img/ 或 posts/2026/		 否（推荐规范填写）
设定自定义域名 （实现极速访问的关键） 配置 jsDelivr 的 CDN 全球加速链接。
固定格式：https://cdn.jsdelivr.net/gh/用户名/仓库名@分支名
例：https://cdn.jsdelivr.net/gh/lijiabao9/pic.cloud@main		 否（但强烈推荐）

💡 为什么要配置自定义域名（CDN 加速）？
GitHub 的官方原始图片存储链接（即 raw.githubusercontent.com 域名）由于众所周知的网络原因，在国内各大运营商环境下的解析极其不稳定，经常出现图片大面积加载失败（碎图）。
强力接入 jsDelivr 的 CDN 加速网络后，它会自动将国内的图片访问请求路由到全球最快的边缘缓存节点，实现网页图片的秒加载。

配置填写完毕后，点击 确定 按钮，并在“图床设置”中将当前配置点击设置为 默认图床。

第四步：打造行云流水的极速写作工作流

当基建全部铺设就绪后，你可以配置以下高能操作，体验接近零摩擦的现代化 Markdown 写作：

1. 黄金搭档：快捷键截图直传

推荐配合国民级截图神器 Snipaste 一起使用，体验绝佳：

  1. 按下 F1（Snipaste 默认快捷键）截取屏幕任意区域，随后按下 Ctrl + C 将截图一键送入系统剪贴板。
  2. 此时，按下 PicGo 的系统级上传快捷键（可在 PicGo 设置中修改为你习惯的组合键，如 Ctrl + Shift + G）。
  3. 软件在后台会自动完成：读取剪贴板 -> 自动上传到 GitHub -> 通过 jsDelivr 生成加速链接。
  4. 听到上传成功的音效后，直接在你的 Markdown 编辑器（如 Typroa、Obsidian）中按下 Ctrl + V，带有标准 ![]() 格式的在线图片就已经完美插入文章中了。

2. 自动化进阶配置

为了防止多设备、多图片上传由于重名导致被 GitHub 拒绝覆盖，强烈建议在 PicGo 设置 中开启以下开关：

  • [√] 时间戳重命名：上传时会自动将文件名转化为时间戳微秒，从根源解决文件名冲突。
  • [√] 上传后自动复制 URL：保证上传成功后你的剪贴板第一顺位永远是合规的在线图片链接。

3. 体积优化：全自动 WebP 压缩

为了极致压榨博客的加载速度，我们可以在 PicGo 的 插件设置 中，搜索并安装名为 webp 的专属图像处理插件。

  • 安装并启用该插件后，你在本地截取的任何大体积 .png 或 .jpg 图片，在上传至云端的瞬间都会被自动在内存中无损压缩并转换为 .webp 格式，图片体积平均可以骤降 60% - 80%，让你的图床仓库更加轻量和长寿。

结语：图床终极方案大功告成！

至此，这套完全属于你个人的、完全免费的全球加速图床已经彻底搭建完毕。

通过这套方案，你的每一张图片都妥善保存在了你自己的 GitHub 账号下，数据所有权完全由你掌控，再也不必忍受第三方商业图床昂贵的流量计费或猝不及防的防盗链限制。

接下来，尽情享受这套“一键截图、一键上传、一键粘贴”的无缝写作体验吧！
