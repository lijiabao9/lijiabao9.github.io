GitHub SSH 连接配置与签名验证

发表于2026-06-18|更新于2026-06-30|运维实施
|总字数:829|阅读时长:3分钟|浏览量:

通过 SSH 方式连接 GitHub，不仅比 HTTPS 更安全，还能通过配置签名密钥让每次提交显示 Verified 绿色标识，提升代码可信度。

本文使用更安全的 ed25519 算法，完整涵盖密钥生成、公钥添加、连接验证及本地签名配置。

1. 生成 SSH 密钥对

推荐使用 ed25519 算法，性能与安全性均优于 RSA。

📌 邮箱建议：

  • 推荐使用与 GitHub 账户一致的邮箱，提交会自动关联
  • 若不想暴露真实邮箱，可在 GitHub Settings → Emails 中开启 Keep my email addresses private，系统会分配匿名邮箱，格式为：你的ID+用户名@users.noreply.github.com

执行后系统会询问：

提示 建议操作
保存密钥位置 直接按 Enter，接受默认路径 ~/.ssh/id_ed25519
密码短语（Passphrase） 可选。设置后每次使用需输入，增强安全性；跳过则更方便

2. 添加公钥到 GitHub

复制公钥内容

添加认证密钥（Authentication Key）

  1. GitHub 右上角头像 → Settings
  2. 左侧菜单 → SSH and GPG keys
  3. 点击 New SSH key
  4. 填写信息：
字段 说明
Title 自定义标识，如 My-PC
Key type 选择 Authentication Key
Key 粘贴公钥内容
  1. 点击 Add SSH key

此密钥用于 git clone、git push、git pull 等仓库操作的身份认证。

添加签名密钥（Signing Key）

重复以上步骤，但 Key type 选择 Signing Key。

此密钥用于验证本地提交签名，使其在 GitHub 上显示 Verified 绿色标识。

4. 配置本地提交信息

设置用户信息

邮箱建议使用 GitHub 分配的匿名邮箱，避免暴露真实地址。

配置 SSH 签名

配置说明

配置项 作用
gpg.format ssh 设置签名格式为 SSH（而非 GPG）
user.signingkey 指定用于签名的公钥路径
commit.gpgsign true 每次 git commit 自动签名，无需手动加 -S

5. 验证签名效果

完成配置后，正常执行 git commit 即可自动签名。

推送至 GitHub 后，提交记录旁会显示 Verified 绿色标识 ✅ Verified。

常见问题

Q：Authentication Key 和 Signing Key 可以共用同一把密钥吗？
A：可以。同一把 ed25519 密钥对既可用于认证，也可用于签名，按上述步骤分别添加即可。

Q：已有 RSA 密钥，需要重新生成吗？
A：非必须。但 ed25519 安全性更高且性能更好，建议迁移。

Q：自动签名后如何临时跳过？
A：执行 git commit --no-gpg-sign 可跳过本次签名。

Q：如何查看当前签名配置？
A：执行 git config --list | grep -E "(sign|gpg)" 查看相关配置。
