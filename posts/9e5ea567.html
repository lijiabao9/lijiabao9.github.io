Hexo 博客入门：从零搭建

发表于2026-05-22|更新于2026-05-30|创意工坊建站日志
|总字数:1.8k|阅读时长:6分钟|浏览量:

Hexo 是一款快速、简洁且高效的静态博客框架。

本文将手把手带你从零开始搭建一个属于自己的静态博客， 涵盖本地环境配置、GitHub Pages 免费托管、GitHub Actions 自动流水线部署以及必装的 SEO 插件推荐， 助你打造高颜值的个人技术站点。

本文基于最新版本的 Hexo 8.1.2 进行演示。

💡 官方资源：

  • 官方网站：https://hexo.io/zh-cn/
  • GitHub 仓库：https://github.com/hexojs/hexo

环境要求

在开始之前，请确保你的电脑已安装以下基础环境：

  • Git

    • 版本要求：无特殊要求
    • 演示版本：git version 2.54.0.windows.1

  • Node.js

    • 版本要求：Hexo 7.0+ 版本要求 Node.js >= 14.0.0
    • 演示版本：v24.15.0

6. 一键部署到 GitHub Pages

本地预览完成后，我们需要将博客部署到外网。GitHub Pages 提供了免费且稳定的静态托管服务。

6.1 准备仓库

  1. 登录 GitHub。
  2. 创建一个 Public（公开） 仓库，仓库名必须严格命名为：用户名.github.io（例如：lijiabao9.github.io）。

6.2 配置 SSH 公钥

为了让本地终端拥有向 GitHub 推送代码的权限，需配置 SSH Key：

  1. 打开终端执行：ssh-keygen -t ed25519 -C "your_email@example.com"，一路回车。
  2. 前往 ~/.ssh/ 目录找到 id_ed25519.pub，复制其中的全部内容。
  3. 进入 GitHub → Settings → SSH and GPG keys → New SSH key。
  4. Title 随意填写，类型选择 Authentication Key，Key 粘贴刚才复制的内容，点击保存。

7. 进阶：GitHub Actions 自动流水线部署

每次写完文章都手动执行 hexo d 效率较低。我们可以利用 GitHub Actions 实现“提交源码，云端自动打包发布”的自动化工作流。

7.1 创建源码私有仓库

  1. 在 GitHub 上新建一个 Private（私有） 仓库，命名为 hexo-blog-source（用于存储博客源码）。
  2. 将本地整个博客项目关联到该仓库并推送到远程：

7.2 配置部署 Token（关键步骤）

为了让 Action 有权限跨仓库推送打包后的静态页面到 Pages 仓库，需要授予权限：

  1. 生成 Token：前往 GitHub → Settings → Developer Settings → Personal access tokens → Tokens (classic) → Generate new token (classic)。
    • 勾选 repo 和 workflow 权限。
    • 生成后，复制保存该 Token（以 ghp_ 开头，仅显示一次）。
  2. 配置到源码仓库：
    • 进入你的 hexo-blog-source 源码私有仓库 → Settings → Secrets and variables → Actions → New repository secret。
    • Name 填写：HEXO_DEPLOY_TOKEN。后续配置会用到此名称。
    • Secret 填写：刚才复制的 Token 字符串。
