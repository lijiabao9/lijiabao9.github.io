Oracle Cloud 免费对象存储配置与 OpenList 挂载指南

发表于2026-06-18|更新于2026-06-28|图床搭建
|总字数:696|阅读时长:2分钟|浏览量:

20GB 免费容量 + 5 万次 API 请求/月，S3 兼容协议挂载至 OpenList

免费政策

资源 免费额度
存储容量 总计 20GB（标准/低频/归档）
API 请求 每月 5 万次

1. 创建存储桶

登录 Oracle Cloud 控制台：

  1. 导航菜单 → 存储 → 对象存储与归档存储 → 存储桶
  2. 点击 创建存储桶
字段 示例 说明
存储桶名称 bucket-20260101-1111 自定义，建议包含日期便于管理
存储桶范围 名称空间 默认
默认存储层 标准 根据需求选择

确认后点击 创建存储桶。

2. 获取关键信息

存储桶信息

进入存储桶详情页，复制以下信息：

信息 示例
存储桶名称 bucket-20260101-1111
名称空间 xxxxxxxxxxxxx

生成访问密钥

点击控制台右上角头像 → 用户设置 → 令牌和密钥 选项卡：

  1. 在 客户密钥 区域，点击 生成密钥
  2. 名称：任意，如 OpenList-Uploader
  3. 复制生成的 密钥值（Secret Value）：
  4. 在密钥列表中，点击右侧 三个点 → 复制 访问密钥（Secret Id）：

⚠️ 密钥值仅显示一次，请妥善保存。丢失需重新生成。

3. 获取区域 ID

控制台上方，头像左侧显示当前区域，如 Japan Central (Osaka)。

参考 Oracle 官方区域文档，在 Table 2-1 中找到对应的 Identifier：

区域名称 区域 ID
Japan Central (Osaka) ap-osaka-1

4. 构建端点地址

端点格式

替换变量

变量 值
<namespace> xxxxxxxxxxxxx
<region> ap-osaka-1

完整端点示例

5. OpenList 挂载配置

在 OpenList 后台添加挂载：

配置项 值
驱动 对象存储
挂载路径 任意，如 /Oracle-Object-Storage
存储桶 存储桶名称，如 bucket-20260101-1111
端点 端点地址，如 https://xxxxxxxxxxxxx.compat.objectstorage.ap-osaka-1.oci.customer-oci.com
访问密钥 ID 填入对应的 Secret Id
访问密钥 填入对应的 Secret Value
地区 ap-osaka-1
强制路径样式 勾选开启

6. 常见报错与解决方案

报错 1：证书验证失败

解决方案：在 OpenList 挂载配置中，勾选 强制路径样式。

报错 2：区域不匹配

解决方案：在 地区 字段中填写正确的区域 ID，如 ap-osaka-1。

注意事项

  1. 免费额度限制：20GB 存储 + 5 万次 API 请求/月，超出将计费
  2. 密钥安全：访问密钥仅生成一次，丢失需重新生成
  3. 区域选择：不同区域端点地址不同，请确保区域 ID 正确
  4. 强制路径样式：Oracle Cloud 对象存储需要开启此选项才能正常访问
