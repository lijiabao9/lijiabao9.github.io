Trojan-Go 高性能代理服务器部署与 Nginx 完美回落指南

发表于2026-05-28|更新于2026-05-28|创意工坊网络代理
|总字数:1k|阅读时长:4分钟|浏览量:

Trojan-Go 是 Trojan 协议的一个 Go 语言进阶实现版本，不仅性能强劲，还原生支持多路复用（Multiplexing）。

它的核心防御逻辑是：在外部探测时，它看起来就是一个合法的 HTTPS 网站。

  • 常规 HTTPS 请求（无密码/密码错误）：自动将流量分发（回落）至本地的 Nginx 伪装网站（如个人博客或静态网页）。
  • 合法 Trojan 请求（有密码认证）：解密并建立代理隧道。

一、环境准备与架构确认

在下载二进制文件前，请务必确认服务器的 CPU 架构，避免因架构不匹配导致 Exec format error 错误。

终端输出 对应 CPU 架构 建议下载的包名
x86_64 Intel / AMD 64位 linux-amd64.zip
aarch64 / arm64 ARM 64位 (如 Oracle ARM / 树莓派) linux-armv8.zip

二、下载与安装

以下以 ARM 架构 为例。若你的服务器为 x86_64，请自行将命令中的 armv8 替换为 amd64。

三、服务端配置

1. 生成强认证密码

Trojan 协议的安全基石在于密码。建议生成一个高强度的 Base64 随机字符串：

请妥善保存输出的字符串，后续客户端与服务端均需使用。

2. 编写 Trojan-Go 配置文件

创建并编辑核心配置文件：vi /app/trojan-go/config.json

关键参数详解：

  • local_port: Trojan-Go 监听的对外端口（本例为 2443）。客户端连接时需指定此端口。
  • remote_port: 防探测回落端口。当主动探测或浏览器直接访问 2443 端口且未提供正确密码时，Trojan-Go 会将流量无缝转发至本地的 127.0.0.1:80（即下文配置的 Nginx 伪装站点）。

四、配置 Nginx 伪装站点（完美回落环路）

为了让回落机制生效，我们需要在本地配置 Nginx 监听 80 端口作为伪装。

默认安装 Nginx 即可。

五、Systemd 服务化管理

通过注册系统服务，实现 Trojan-Go 的后台长效运行与开机自启。

1. 创建 Service 描述文件

新建文件：vi /etc/systemd/system/trojan-go.service

2. 激活并启动服务

如果状态输出中显示 Active: active (running)，则说明 Trojan-Go 已经在服务端顺利守护运行。

六、客户端接入指南

在客户端（如 v2rayN, Clash, Nekoray 等）中新建 Trojan 节点，填入以下核心元数据：

  • 协议类型 (Type): Trojan
  • 服务器地址 (Address): ip *
  • 端口 (Port): 2443 (即 config.json 中的 local_port)
  • 密码 (Password): 你生成的强密码
  • 传输层协议 (Network): raw
  • 底层传输安全 (TLS): 开启 (tls)
  • SNI / Peer Name: gfw.yourdomain.com

⚠️ 防火墙安全策略提醒
请务必在云服务器控制台（安全组）或系统自带的 UFW 防火墙中放行 2443 端口，否则客户端将无法建立握手：
ufw allow 2443/tcp
