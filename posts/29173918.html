OracleCloud网络流量解析：如何开放计算实例的端口

发表于2025-06-19|更新于2025-06-21|运维圣殿网络边界
|总字数:1.1k|阅读时长:3分钟|浏览量:

在 Oracle Cloud Infrastructure (OCI) 中，云服务器的网络流量控制涉及多个层级的安全策略，
包括实例级防火墙（iptables/firewalld）、安全列表（Security List）和网络安全组（Network Security Groups, NSG）。

术语缩写：

  • OCI（Oracle Cloud Infrastructure），计算实例，就是云服务器
  • VNIC（Virtual Network Interface Card），虚拟网卡，理解成云服务器的网卡
  • VCN（Virtual Cloud Network），虚拟云网络
  • NSG（Network Security Groups），网络安全组

公网流量到达OCI的过程

在 Oracle Cloud Infrastructure (OCI) 中，即计算实例（服务器）：

VNIC（Virtual Network Interface Card，虚拟网卡）是你的实例（虚拟机）连接到网络（VCN）的“网卡接口”。

  • VNIC 就像一块网卡，它把实例接入虚拟云网络（VCN）。
  • 它决定了实例的 IP 地址、MAC地址（虚拟）、网络安全组、子网 等网络属性。
  • 每个实例至少有一个主 VNIC，你也可以给实例绑定多个 VNIC（多网卡）。

VNIC 决定实例流量的安全规则，因为：
● VNIC 可以绑定一个或多个 网络安全组 (NSG)，控制它的入站/出站流量。
● VNIC 所在的子网也有 安全列表 (Security List)，一起生效。

实例
└── VNIC (虚拟网卡)
├── 私有IP / 公有IP
├── 子网
├── 绑定 NSG (网络安全组)
└── 受子网 Security List 控制

为什么即使是公网 IP，流量也会被 Oracle 的 安全组 / 安全列表 检查？
原因本质：在 Oracle Cloud（以及 AWS、Azure 等公有云），公网 IP 其实是映射，不是直接“绑在”实例上的物理 IP。

🌐 具体过程

  1. 你的实例被分配一个私有 IP（在 VCN 虚拟云网络内）。
  2. Oracle 会给这个私有 IP 映射一个公网 IP，这个公网 IP 是由 Oracle 云平台的 边界网关 或 路由设备 转发到你实例的。
  3. 所以：
    ● 当外部用户访问你的公网 IP：
    👉 数据包先到达 Oracle 的边界路由 / 防火墙设备
    👉 再进入 VCN 的子网 → VNIC → 实例
  4. 在边界路由 / VCN 的过程中：
    ⚠ 流量必须经过：
    ✅ 安全列表（子网级防火墙）
    ✅ 网络安全组（VNIC 级防火墙）

公网流量 → Oracle 云边界设备 → VCN 安全列表 / NSG 检查 → VNIC → 实例（防火墙规则）

实例防火墙管理

Oracle 创建的计算实例，默认写了很多防火墙规则，可以通过命令 iptables -L -n 查看。

由于流量进入实例之前，会经过 安全列表 / 网络安全组 检查，将实例的所有防火墙清除。

但是由于安装了 防火墙规则持久化软件，实例重启后，还会自动写入默认的规则。

通过 iptables 指令管理防火墙太过复杂了，可以安装 apt install -y ufw 工具。
但是规则都是放行了，没必要安装了。

Oracle 安全组/安全列表放行

网络安全组由一组 VNIC 和应用于这些 VNIC 的一组安全规则构成。例如，为 VCN 中具有相同安全状况的所有实例创建一个网络安全组。
与此不同，VCN 安全列表由一组安全规则组成，这些规则应用于与该安全列表关联的子网中的所有 VNIC。

只有一台实例，就不用配置安全组了。

找到 OCI实例 对应的 VCN虚拟云网络，安全 -> 安全列表 -> 安全规则：

默认是禁止了对服务的 ping，如果要开启，把 源是 0.0.0.0/0 的 ICMP 规则，把类型3和代码4去掉；
默认开启了 TCP 的 22 端口，这是远程 SSH 服务用到的；

可以放行 80 端口和 443 端口：
添加入站规则，

  • 源类型，CIDR
  • 源CIDR，0.0.0.0/0
  • 协议，TCP
  • 目的端口，80
