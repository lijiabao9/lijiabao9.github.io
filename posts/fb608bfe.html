OpenList 常用存储挂载配置指南

发表于2026-06-04|更新于2026-06-16|使用教程
|总字数:1.1k|阅读时长:3分钟|浏览量:

对于使用 OAuth2 认证方式挂载的网盘，建议参考 Cloudflare Workers 部署 OpenList API Page 教程 部署令牌获取网站，确保令牌不会泄露。

  • 创建认证应用时，回调地址改需为私有部署地址
  • 网盘挂载时，API地址 和 重定向URL 配置，地址需改为私有部署地址

腾讯微云

基于 Cookie 认证。

  1. 登录 腾讯微云网盘
  2. F12 打开控制台 → Network 标签页
  3. 刷新页面，在请求头中复制 Cookie 完整值
配置项 值
驱动 腾讯微云
Cookie 复制的 Cookie

蓝奏云

支持账号密码或 Cookie 认证，建议使用账号密码方式。

配置项 值
驱动 蓝奏云
类型 账号
账号 手机号/用户名
密码 登录密码

夸克网盘

基于 Cookie 认证。

  1. 登录 夸克网盘
  2. F12 打开控制台 → Network 标签页
  3. 刷新页面，在请求头中复制 Cookie 完整值
配置项 值
驱动 夸克
Cookie 复制的 Cookie

百度网盘

基于 Refresh Token 认证。

创建应用

  1. 登录百度网盘开放平台，进行开发者实名认证后
  2. 创建应用，获取 应用ID (API Key) 和 应用密钥 (Secret Key)。
  3. 配置应用的回调地址，格式为：https://api.oplist.org/baiduyun/callback

获取 Refresh Token

  1. 打开 OpenList 令牌获取工具
  2. 网盘名称选择 百度网盘（OAuth2）验证登录
  3. 填入 Client ID 和 Client Secret → 获取 Token
  4. 授权登录后复制 Refresh Token
配置项 值
驱动 百度网盘
刷新令牌 复制的 Refresh Token
使用在线 API 取消勾选
客户端 ID Google Client ID
客户端密钥 Google Client Secret

GitHub 仓库

基于 Personal Access Token 认证。

  1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. 点击 Generate new token (classic)
  3. 勾选 repo 权限，生成后复制 Token
配置项 值
驱动 Github API
令牌 Personal Access Token
所有者 GitHub 用户名
仓库 仓库名称

Cloudflare R2 存储

基于 S3 兼容协议认证。

创建存储桶

Cloudflare 控制台 → 存储和数据库 → R2 对象存储 → 概述 → 创建存储桶（需绑定信用卡，免费额度内不扣费）

获取 API 凭证

  1. R2 概述 → 账户详情 → API 令牌 → 管理 → 创建 API 令牌
    • 权限：对象读和写
    • TTL：永久
  2. 保存 Access Key ID 和 Secret Access Key
  3. 复制 S3 API 地址（端点）
配置项 值
驱动 对象存储
存储桶 R2 存储桶名称
端点 S3 API 地址
地区 auto
访问密钥 ID Access Key ID
访问密钥 Secret Access Key

谷歌云盘

基于 GCP 凭证及 Refresh Token 认证。

启用 API 与创建凭证

  1. 登录 Google Cloud 控制台
  2. 访问 Google Drive API 快速入门 启用 API
  3. 在控制台，API和服务 → 凭据 → 创建凭据 → OAuth 客户端 ID
    • 应用类型：Web 应用
    • 重定向 URI：https://api.oplist.org/googleui/callback

      若提示 如需创建 OAuth 客户端 ID，您必须先配置权限请求页面，请点击配置同意屏幕按钮，在后续页面中完品牌信息的创建
      受众群体选择 外部用户，填写基础品牌信息后保存返回。

  4. 保存 Client ID 和 Client Secret
  5. 进入 OAuth 客户端详情 → 测试用户 → Add users，添加要挂载的谷歌账号，点击 发布应用

获取 Refresh Token

  1. 打开 OpenList 令牌获取工具
  2. 网盘名称选择 GoogleDrive Login（OAuth2）
  3. 填入 Client ID 和 Client Secret → 获取 Token
  4. 授权登录后复制 Refresh Token
配置项 值
驱动 谷歌云盘
刷新令牌 复制的 Refresh Token
使用在线 API 取消勾选
客户端 ID Google Client ID
客户端密钥 Google Client Secret

OneDrive

基于 Azure 企业应用及 Refresh Token 认证（支持企业/世纪互联/教育账号，个人账号暂不支持）。

注册 Azure 应用

  1. 管理员账号登录 Azure 门户
  2. 新注册：
    • 名称：任意
    • 账户类型：Any Entra ID Tenant + Personal Microsoft accounts
    • 重定向 URL：https://api.oplist.org/onedrive/callback
  3. 复制 Application (client) ID

生成密钥与配置权限

  1. Certificates & secrets → Client secrets → New client secret，复制 Value
  2. API permissions → Add a permission → Microsoft Graph → Application permissions → 勾选 Files.ReadWrite.All
  3. 点击 Grant admin consent

获取 Refresh Token

  1. 打开 OpenList 令牌获取工具
  2. 网盘名称选择 OneDrive（OAuth2）企业账户
  3. 填入 Client ID 和 Client Secret → 获取 Token
  4. 授权登录后复制 Refresh Token
配置项 值
驱动 OneDrive
刷新令牌 复制的 Refresh Token
使用在线 API 取消勾选
客户端 ID Azure 客户端 ID
客户端密钥 Azure 客户端密钥
