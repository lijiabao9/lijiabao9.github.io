Hexo 博客快速搭建指南

发表于2025-06-10|更新于2025-06-16|项目工坊造物日志Hexo博客
|总字数:1.5k|阅读时长:5分钟|浏览量:

想要快速搭建一个轻量、高效的静态博客？
Hexo 凭借其简洁的架构和丰富的插件生态，成为许多写作者与技术爱好者的首选。
本指南涵盖安装、初始化、文章撰写等核心步骤，助你轻松开启博客之旅。

欢迎使用 Hexo！

  • 超快速度：Node.js 所带来的超快生成速度，上百个页面在几秒内完成渲染
  • 支持Markdown：Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至整合 Octopress 的大多数插件
  • 一键部署：只需一条指令，即可部署到 GitHub Pages，Heroku 或其他平台
  • 插件：强大的 API 带来无限的可能。支持数种模板引擎（EJS、Pug、Nunjucks等）。可以与现有的 NPM 包轻松地集成

这是您的第一篇文章。查看 文档 获取更多信息。
如果您在使用Hexo时遇到问题，可以在 故障排查 中找到答案，
或者在 GitHub 提问。

环境要求

  • Git
  • Node.js (Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本)
  • 安装 Hexo，通过指令 npm install hexo-cli -g

创建项目

初始化一个项目，如果没有指定 folder，默认在当前目录下进行。
本命令，相当于执行了

  1. git clone hexo-starter 和安装 hexo-theme-landscape 主题
  2. 通过 npm 包管理器下载依赖

项目文件夹目录如下：

  • scaffolds 存放模板文件
    hexo new layout :title 指令的 layout 参数指定模板
    draft 草稿，page 页面，post 文章
  • source 资源文件夹
    开头命名为 _ 开始的文件/文件夹，将会跳过不处理（除了 _posts，是存放文章的位置）。
    Markdown 和 HTML 文件会被解析渲染后放到 public 文件夹（类似于编译后的 dist），其他文件会被直接拷贝。
  • themes，主题文件夹
    安装的主题文件，会被放到这里
  • _config.yml，网站配置文件
    在这里配置参数。详细参数解释，查看官方文档 配置
  • package.json，应用程序信息
    默认安装了hexo所需的依赖，如 EJS、Stylus、Markdown 渲染引擎等。

部署配置（hexo deploy）

需要到 GitHub 上创建一个 public 的仓库，仓库名称格式：<用户名>.github.io。
到仓库的 settings -> Pages，在 Build and deployment 的 Branch 中，将 None 改为 master 分支，将仓库配置为 GitHub Page。
例如我的仓库：lijiabao9.github.io。
部署后，就可以通过网址 Gabriel的博客 访问了。

hexo deploy 部署指令，需要安装部署插件和指定部署信息

  1. npm install hexo-deployer-git --save
  2. 指定部署信息，_config.yml

本地电脑运行部署指令时，会弹出要求输入 GitHub 的账号密码认证。
可以在本地配置生成 SSH密钥，将密钥放到 GitHub 账号上，就不用每次都要验证了。

  1. ssh-keygen -t rsa -C "you email@example.com"
    连续3次回车默认配置后，在本地电脑的用户目录下，会生成公钥 ~/.ssh/id_rsa_pub 和私钥 ~/.ssh/id_rsa。
  2. 将公钥文件放到 GitHub 账号
    到 GitHub 主页的 Settings -> SSH and GPG keys 处， 点击 Add new SSH key，
    配置 Title 随便起名；
    配置 Key type 保持默认 Authentication Key；
    配置 key，将上面的公钥文件的内容粘贴进来；

自动部署（GitHub Action）

通过上述的插件进行部署时，会在本地编译打包成静态文件后，才会将静态文件推送到 GitHub Page 仓库。
这样的问题是，随着后续文章等资源越来越大，编译打包的速度会越来越慢。

可以通过 GitHub Action 实现自动部署，
思路是将博客源码托管到 GitHub 的另一个私库，给这个私库配置 Action 执行打包推送等操作，
后续就只用管理和提交博库项目的源码即可，无需关注部署。

  1. GitHub 新建一个博客项目源码的私库，如： hexo-source-Repo
  2. 由于 GitHub Action 需要 token 授权，生成一个 token
    主页 Settings -> Developer settings -> Personal access tokens -> Token(classic) 处，
    右上角 Generate new token，选择 Generate new token（classic），
    配置 Note 随便起名；
    配置 Expiration 选择永不过期（No expiration）；
    配置 Select scopes，勾选 repo 和 workflow；
    生成后，将 token 复制保存（只会显示一次，丢失了需要成功生成新的）
  3. 在 hexo 项目中，新增工作流配置文件 /.github/workflows/autodeploy.yml

    将配置文件的 部署到Github 配置信息修改为自己的，
    token 是上述生成的，repository-name 是 GitHub Page 仓库地址。
    所有的分支默认使用 master。

  4. 将本地项目与远程仓库关联
