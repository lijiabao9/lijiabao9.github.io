Windows 环境下 MySQL 绿色压缩包版安装与配置指南

发表于2026-05-23|更新于2026-05-25|开发环境Windows
|总字数:1.4k|阅读时长:4分钟|浏览量:

相比于传统的 MSI 安装包，使用 ZIP 压缩包（Archive）方式部署 MySQL 具有明显的优势：系统侵入性极低、配置高度灵活、且卸载时干净无残留。

本文将为你完整记录在 Windows 环境下从零开始部署 MySQL 9.7.0 绿色版的标准流程。

一、 下载与解压

1. 获取官方压缩包

前往 MySQL 官方下载页面，选择 Windows (x86, 64-bit), ZIP Archive 版本进行下载。

官方下载地址：https://dev.mysql.com/downloads/mysql/

2. 解压与部署

将下载好的压缩包解压到你指定的非系统盘目录下（路径中尽量避免包含中文或空格）。

  • 示例路径：D:\App\MySQL\mysql-9.7.0\

二、 核心配置 (my.ini)

MySQL 在初始化和启动时需要读取核心配置文件。

六、 进阶建议：配置全局环境变量

为了避免每次使用 mysql 命令都必须繁琐地进入 bin 目录，强烈建议将其配置到系统的环境变量中：

  1. 右键点击 此电脑 -> 选择 属性 -> 点击 高级系统设置 -> 点击 环境变量。
  2. 在 系统变量 区域中找到并选中 Path 变量，点击 编辑。
  3. 点击 新建，将你的 MySQL bin 目录完整路径添加进去：D:\App\MySQL\mysql-9.7.0\bin。
  4. 一路点击 确定 保存退出。

配置完成后，打开任意一个新的终端窗口，你都可以直接输入 mysql -u root -p 随时随地登录数据库。

七、 彻底卸载与清理

如果未来因为迁移或升级需要卸载该版本的 MySQL，压缩包版的卸载流程极为纯粹和干净，只需简单四步：

  1. 服务清理
  2. 物理删除：直接将整个 mysql-9.7.0 安装文件夹从磁盘上彻底删除。
  3. 清理环境：如果你配置了步骤六中的 Path 环境变量，回到设置中将该条路径删掉即可。

至此，你的操作系统将不留任何残留，干净如初。
