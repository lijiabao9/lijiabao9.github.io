升级 package

后续项目有改动的话，

npm version 后面参数说明：

patch：小变动 0.0.1->0.0.2
minor：增加新功能 0.0.1->0.1.1
major：大版本更新 0.0.1->1.0.1

删除 pacakge

注意，删除后，24 小时后，才可重新发布同名的包。

而且有一个坑的地方，重新发布的版本号，不可与之前使用过的重复，避免与缓存冲突。
否则会报错。

npm ERR! code E400
npm ERR! 400 Bad Request - PUT https://registry.npmjs.org/hello-world-gabriel - Cannot publish over previously published version “1.0.0”.

静态资源 cdn 加速

发布到 npm 上的 package，里面如果有静态资源，
默认可以通过 jsDelivr 加速访问，

URL 格式：

实际例子：
https://cdn.jsdelivr.net/npm/hello-world-gabriel/index.js
