彻底打通甲骨文云 (OCI) Ubuntu 实例：双重防火墙与 SSH 权限优化指南

发表于2026-05-24|更新于2026-05-24|DevOps云服务器
|总字数:1.5k|阅读时长:4分钟|浏览量:

在使用 Oracle Cloud (甲骨文云) 的 Ubuntu 实例时，新手常遇到的最大门槛就是 “双重防火墙”：

  1. 云端防火墙：OCI 控制台的安全列表 (Security Lists) 或网络安全组 (NSG)。
  2. 系统防火墙：Linux 系统内部的 Netfilter/IPTables。

只有两层防火墙都放行，外部流量才能真正到达你的服务。本文将带你一站式打通网络配置，并附带开启 Root 密码登录的优雅实现。

第一层：云端防火墙 (Security Lists)

这是 Oracle 设定的第一道关卡（云端虚拟防火墙）。默认情况下，OCI 处于安全考虑，仅放行了 SSH (22 端口)。

配置步骤

  1. 登录 OCI 控制台，进入 实例详情 页面。
  2. 选择 网络 (Network) 选项卡中的 子网 (Subnet) 名称链接。
  3. 在子网详情页，选择 安全 选项卡，点击 安全列表 (Security Lists) 中默认的 Default Security List for...。
  4. 在安全列表详情页，选择 安全规则 选项卡，点击 添加入站规则 (Add Ingress Rules)。

推荐规则 (将控制权下放给系统)

如果你希望由服务器内部防火墙接管所有流量控制，或者用于测试环境，可以添加一条“全放行”规则：

配置项 填入内容 含义
源 CIDR 0.0.0.0/0 允许来自全球任何 IP 的访问
IP 协议 所有协议 放行 TCP、UDP、ICMP 等所有流量
源端口范围 (留空) 放行所有源端口

附录：Oracle 网络术语解析

在配置甲骨文云的网络时，频繁出现的各种缩写常让新手摸不着头脑。理解它们有助于你未来构建更复杂的云上多机内网架构。

💡 核心概念速查手册

  • OCI (Oracle Cloud Infrastructure)：即甲骨文云的基础设施平台整体简称。
  • VCN (Virtual Cloud Network)：虚拟云网络。你在云上划出的一块绝对隔离的私有网络区域，类似于你家里的独立局域网。
  • VNIC (Virtual Network Interface Card)：虚拟网卡。实例连接到 VCN 的媒介，它决定了实例的私有 IP 以及绑定的公网 IP。一个实例可以绑定多个 VNIC 满足多网卡需求。
  • Security List (安全列表)：子网级别的防火墙。作用于该子网内的所有网卡，通常用于配置大范围、通用的放行规则。
  • NSG (Network Security Groups)：网卡级别的防火墙。比安全列表更灵活，直接精确绑定到具体的 VNIC 上，实现单机单况的精细化控制。
