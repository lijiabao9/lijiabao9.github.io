使用ACME和CloudFlare实现HTTPS证书自动申请与续期

发表于2025-06-12|更新于2025-06-23|运维圣殿网络边界
|总字数:1.2k|阅读时长:4分钟|浏览量:

本文介绍如何利用 ACME 协议（如 Let’s Encrypt）配合 Cloudflare API Token 实现全自动的 SSL/TLS 证书申请与续期流程。
通过本文的指导，您可以为自己的网站部署全自动的 HTTPS 证书管理方案，告别手动续期证书的烦恼，同时确保网站始终使用最新的安全证书。

网站服务升级为 HTTPS 协议，需要拥有一个域名，以及向 CA 机构申请域名对应的证书。

假设已经拥有域名 example.com ，可以将域名托管到 CloudFlare 网站。
对域名开启了 CF 的代理（橙云）后，可以享受 CF 提供的 全球CDN 加速，CF 还免费提供了有效期为 15 年的证书。
CF 的机制是，客户端 —-(边缘证书)— CF —-(源证书)—- 服务器

但是 CF 提供的源证书，不是可信证书，只对 CF 站点起效。
当我们的域名关闭 CF代理（即变为 灰云），域名解析会直接返回服务器的真实IP地址，不再是 CF 的 CDN 节点地址。
这个时候，CF 的源证书，就不起作用了，因为它不是可信的证书。

这就需要主动去向可信的 CA 机构申请证书了。
但是一般申请下来的证书，有效期只有 90 天，到期了又需要重新去申请，这就很麻烦。

下面介绍一种基于 ACME 和 CloudFlare API Token 的方式，实现证书的自动申请与续期。

ACME，Automated Certificate Management Environment，自动化证书管理环境
是一种协议，用于自动化申请、验证和管理 SSL/TLS 证书，
最著名的实现是 Let’s Encrypt（一个免费、自动化的证书颁发机构）。

申请证书时，有 3 种验证方式：

  1. 邮箱验证
    CA 机构会向你发送一个电子邮件，需要点击邮件内的链接完成验证。
  2. 文件验证（HTTP-01挑战）
    CA 机构会要求你在服务器的指定路径上放一个文件（内容他们随机指定），然后开放 80 端口让 CA 机构验证；
  3. DNS记录验证（DNS-01挑战）
    需要你在域名的 DNS记录 中，添加一条 TXT 记录，CA 机构会去验证；

我们用到的是第三种方式。
并且利用 CloudFlare 提供的 api 功能，授权 ACME 自动添加 DNS记录，实现自动化。

首先，需要在 CloudFlare 生成一个 Token。
个人主页（右下角）-> 管理账号 -> 账户API令牌 -> 创建令牌
使用API令牌模板，选择 编辑区域DNS，说明这个令牌只对指定的域名起效，更安全。

令牌名称：ACME自动证书（example.com）
权限：区域，DNS，编辑
区域资源：包括，特定区域，example.com

创建成功后，令牌只会显示一次，复制保存：token_xxxxxxxxxxxxxx

然后，登录服务器，到用户主目录下：

把 Token 写入到 .bashrc 中
export CF_Token=token_xxxxxxxxxxxxxx
保存后，执行命令 source .bashrc 刷新用户变量。

重新连接终端后，执行证书申请操作：

设置CA机构为 letsencrypt，默认的 zerossl 需要注册账号
acme.sh --set-default-ca --server letsencrypt
提交证书申请，（泛域名证书）
acme.sh --issue --dns dns_cf -d example.com -d "*.example.com"

执行成功后，会看到如下的输出：
Your cert is in: /root/.acme.sh/example.com_ecc/example.com.cer
Your cert key is in: /root/.acme.sh/example.com_ecc/example.com.key
