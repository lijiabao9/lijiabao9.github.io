在 Hexo Butterfly 主题中优雅集成 Umami 隐私分析统计

发表于2026-05-25|更新于2026-05-28|创意工坊建站日志
|总字数:882|阅读时长:3分钟|浏览量:

Butterfly 主题原生支持 Umami 统计，配置流程非常简单。

通过集成 Umami，你不仅可以摆脱传统不蒜子（Busuanzi）偶尔断流或加载缓慢的问题，还能完全将数据自主权握在自己手中。

本文将带你一步步在 Butterfly 主题中启用 Umami，并解锁“硬核”的线上 PV/UV 计数器展示功能。

一、修改主题配置文件

打开你的 Hexo 博客根目录，编辑 Butterfly 主题的配置文件 _config.butterfly.yml：

1. 关闭原生不蒜子统计（可选）

为了避免页面上出现重复的计数器或导致加载冗余，建议先关闭不蒜子统计：

2. 启用并配置 Umami Analytics

在配置文件中找到 umami_analytics 块（若没有可直接复制以下内容追加到文件末尾），并填入你的自建服务信息：

二、获取 Access Token (关键步骤)

Butterfly 主题为了在博客底部和文章内页实时调用 Umami 的 API 接口来渲染访问量，需要配置一个具备读权限的 Authorization Token。

对于自托管（Self-hosted）版本的 Umami，目前有以下两种主流的获取方式：

方法 A：开发者工具抓取法（最推荐，简单快捷）

  1. 使用浏览器（如 Chrome、Edge）登录你的 Umami 后台管理系统。

  2. 登录成功后，按下 F12 键打开开发者工具，并切换到 Network (网络) 面板。

  3. 手动刷新一下 Umami 后台页面，此时网络面板会抓取到很多请求。

  4. 随意点击一个 API 请求（例如名字为 stats、websites 或 metrics 的请求）。

  5. 在右侧的 Headers (请求头) -> Request Headers 中，找到 Authorization 字段。

  6. 复制 Bearer 后面那一长串由字母和数字组成的随机字符串。注意：不要复制 Bearer 单词本身和空格。

方法 B：API 接口模拟登录（极客推荐）

如果你熟悉终端或 API 测试工具，可以通过 cURL 直接向你的 Umami 实例发送登录请求：

执行后，服务器会返回一个 JSON 对象，其中 token 字段对应的值即为你需要填入主题配置的信息。

📌 安全提示：请妥善保管好你的 Token，切勿将包含真实 Token 的配置文件直接开源到公共 Git 仓库中。如需开源，建议使用环境变量进行敏感信息分离。

三、博客重新编译发布

同时，登录你的 Umami 后台，也能实时观测到访客的足迹。
