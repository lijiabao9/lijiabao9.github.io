Ubuntu 部署 Java 环境通常有两种方式：

  1. apt 安装（推荐）：最简便，自动处理依赖与更新，安装的是 OpenJDK。
  2. 手动解压安装：适合特定版本（如 Oracle JDK）或需要精确控制安装路径的场景。

3. 环境变量加载机制

  • /etc/profile：全局配置，用户登录时加载一次
  • ~/.profile：用户级配置，登录时加载一次
  • ~/.bashrc：用户级配置，每次打开新终端加载

生产服务器建议修改 /etc/profile 确保所有系统服务共享同一套 Java 环境。自动化脚本（如 Jenkins、Crontab）属于 Non-login Shell，可能跳过 /etc/profile，稳妥的做法是在 /etc/bash.bashrc 也追加一份。
