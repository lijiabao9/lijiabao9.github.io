3X-UI 进阶搭建高隐蔽性代理系统指南

发表于2026-05-28|更新于2026-06-14|网络代理
|总字数:908|阅读时长:3分钟|浏览量:

3X-UI 是 X-UI 的优秀分支，支持 Xray 内核的 Reality 等新特性，并提供美观的数据统计面板。

本文介绍兼顾管理便捷性与高隐蔽性的代理系统搭建。

1. 安装与初始化

安装完成后终端输出随机生成的初始登录信息。

系统优化

终端输入 x-ui 进入管理菜单：

  • 输入 23 安装 BBR 拥塞控制算法
  • 输入 6 修改管理员账号和密码

2. 面板安全加固（Nginx 反代）

将面板隐藏在 Nginx 443 端口下，通过特定路径访问。

修改面板监听

登录面板 → 面板设置 → 常规配置：

  • 面板监听 IP：127.0.0.1（仅本机可访问）
  • 面板监听端口：比如 20443
  • 面板 URL 根路径：/xui/（前后均有斜杠）
  • 保存并重启面板

重启后无法通过 IP:端口 直接访问面板，需完成 Nginx 配置。

开启订阅设置

登录面板，面板设置 -> 订阅设置：

  • 启用订阅服务：勾选开启
  • 监听端口：比如 20444
  • URL 路径：比如 /subscribe/xxx-xxxx-xxxx-xxx/

Nginx 配置

创建 vi /etc/nginx/sites-available/gfw.yourdomain.com：

访问 https://gfw.yourdomain.com/xui 即可访问登录面板。

3. 方案 A：VLESS + WS + TLS（CDN 模式）

Nginx 统一处理 TLS，转发 WebSocket 流量至 Xray，可接入 Cloudflare CDN。

面板配置

添加入站：

配置项 值
备注 VLESS-WS-TLS
协议 vless
地址 127.0.0.1
端口 20445
传输协议 WebSocket
路径 /ray
外部代理 TLS + gfw.yourdomain.com + 443
安全 无（TLS 由 Nginx 处理）
  • 面板安全加固的 Nginx 配置中，已经对 /ray 做转发处理
  • 外部代理配置，是为了生成的节点地址正常可用

4. 方案 B：VLESS + Reality（直连性能模式）

无需域名和证书，直接借用大厂 TLS 证书，传输效率高、延迟低。

面板配置

添加入站：

配置项 值
备注 VLESS-Reality-Vision
协议 vless
监听 IP 留空
端口 20446
传输协议 RAW
外部代理 相同 + 服务器IP + 20446
安全 reality
uTLS chrome
目标 www.tesla.com:443
SNI www.tesla.com
公钥/私钥 点击 获取新证书 自动生成

5. 方案对比

模式 路径 优势 适用场景
VLESS + WS + TLS 客户端 → CDN → Nginx(443) → Xray 可套 CDN 隐藏 IP 日常浏览、IP 被墙备用
VLESS + Reality 客户端 → Xray(直连端口) 无需域名证书、延迟极低 视频、下载、游戏等高吞吐场景
