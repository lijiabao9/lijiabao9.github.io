从购买到配置：手把手教你将域名托管到Cloudflare

发表于2025-06-12|更新于2025-07-09|运维圣殿网络边界
|总字数:1.3k|阅读时长:4分钟|浏览量:

本文详细介绍如何购买域名并托管至 Cloudflare 。
无论你是个人站长还是企业运维，本文都能帮你高效管理域名并提升网站安全性与访问速度！

购买域名

选择 SpaceShip 购买顶级域名 .xyz，目前6位数字的域名，有优惠。

注册账号后，添加支付方式（可选择 Visa信用卡、支付宝 等），账单地址建议填写真实的邮箱，后面用来接收消息。

网站首页，搜索 6位数字.xyz 的域名，比如 131499.xyz。
只要搜出来的域名是可用的，且价格显示是 $0.67 的，才要购买，否则续费价格会很昂贵的。

购买域名成功后，只有一年使用权，去续费弄成十年的。
网站顶部的 Launchpad -> Domain Manager，点击选中对应的域名；
点击 Extend subscription，直接选择 续费9年，需要付费 $6.03。

就这样，50RMB 不到，就可以拥有一个十年的域名了。

域名托管到 CloudFlare

由于国内域名需要实名备案，选择将域名托管到 CloudFlare。

注册账号后，账号主页 -> 添加域，输入购买到的域名 131499.xyz，选择 快速扫描 DNS 记录；
选择 免费计划（Free），扫描出来的 DNS 记录，全部删除掉，点击继续激活；

这时需要修改域名的 NS（Name Server Record），用于指定域名是由哪个 DNS 服务器来进行解析。
到 spaceship网站顶部的 Launchpad -> Domain Manager，点击选中对应的域名，选择 Nameservers & DNS，
将名称修改成刚才 cloudflare 页面提供的 2 个 NS；

回到 cloudflare 页面，点击 继续，check nameservers now让服务器检查 NS 的变更状态，
通常需要 10 分钟或更长，直到主页的对应域名的状态，变成 活动Active（会收到邮件通知激活成功）。

域名托管配置

重点是 DNS -> 记录 选项，在这里对域名添加记录：
一般添加的都是 A 类型的记录，将域名解析到指定的 ipv4 地址；
记录可以添加多个，对用到的 三级域名，添加解析；
例如：

类型 名称 IPv4地址 代理状态
A @ 11.11.11.11 橙云（代理）
A www 11.11.11.11 橙云（代理）
A zhidao 11.11.11.11 灰云（仅DNS）

名称是指 三级域名的名称，比如 www 对应的域名是 www.131399.xyz，@ 是特殊的表示二级域名 131399.xyz。
代理状态，如果开启了代理，表示启用 CF 的 CDN，客户端和源服务器通过 CDN 进行中转，隐藏源服务器；
代理状态，仅 DNS 是指，CF 会将域名的 真实IP 返回，客户端直接访问源服务器；

如果开启了橙云，根据需求可配置以下：

  • DNS -> 设置，开启 DNSSEC、多签名者DNSSEC
  • 电子邮件，DMARC管理，开启该功能
  • 安全性 -> 设置，管理检测，开启 自动程序流量 和 客户端滥用 的所有
  • 速度 -> 优化，启用所有可用配置，将所有的都开启
  • 缓存 -> 配置，开启 CSAM扫描工具、Crawler Hints、Always Online
  • 规则 -> 设置，标头托管转换、标准化URL 开启所有
  • 网络，除了 Pseudo Ipv4，其他都开启
  • Scrape Shiedld，开启 电子邮件地址混淆技术、Hotlink保护（这个开启后，别的网站就不能引入本站的图片了）

还有一个重点的就是 SSL/TLS 配置：
橙云模式下，客户端 —-(边缘证书)—- cloudflare —-(源证书)—- 服务器

在 源服务器，可以创建证书：

  • 使用 Cloudflare 生成私钥和 CSR，私钥类型 RSA（2048）
  • 列出源服务器上应受证书保护的主机名： *.131499.xyz 131499.xyz
  • 证书有效期：15年

复制保存 源证书（131499.xyz.pem）和 私钥(131499.xyz.key)。私钥只会显示一次。
这个证书就可以在网站上配置使用了，但是只对 cloudflare 可信。
如果想要生成完全可信的证书，可以参考 使用ACME和CloudFlare实现HTTPS证书自动申请与续期

经过身份验证的源服务器拉取。需要在 Web 服务器上进行其他配置，防止客户端绕过 Cloudflare，直接向源服务器发送请求。

这时候，概述的加密模式，可以改成 完全（严格）。

边缘证书，这是 浏览器 和 CloudFlare 之间的通信配置：

  • 始终使用 HTTPS，将 http 请求自动重定向到 https
  • HTTP严格传输安全（HSTS），要求浏览器强制执行安全策略。需要网站开启 https
  • 最低 TLS 版本，默认保持 1.0
  • 随机加密，默认开启。随机加密可以让浏览器知道您的站点通过加密连接提供
  • TLS 1.3，默认开启
  • 自动HTTPS重写，默认开启。当网站引入外站的资源，自动尝试 https 获取
  • 开启 证书透明度监视。证书颁发机构为域名颁发证书时，将收到一封电子邮件通知
  • 禁用通用SSL。删除边缘证书
